────────────────────────────────────────────────────────────────────────────────
  NOTIFICATION TYPES SUPPORTED
────────────────────────────────────────────────────────────────────────────────

  Type "-1"  →  Browser Push (native OS notification via Service Worker)
  Type "0"   →  In-App Toast (slide/fade card via createVariant + renderToast)
  Type "1"   →  Custom Template (user-provided JS compiled via new Function())
  Type "2"   →  Glassmorphism HTML (frosted-glass card via showHtmlNotification)


────────────────────────────────────────────────────────────────────────────────
  RENDERING ENGINES
────────────────────────────────────────────────────────────────────────────────

  Type 0:  variants.js → createVariant()  +  default.js → renderToast()
           - Toast card with icon/logo, title, description, media, buttons
           - Dark mode via system preference (CSS + JS matchMedia)
           - Stacking via toastStack container (bottom-left/right)

  Type 2:  html-notification.js → showHtmlNotification()
           - Glassmorphism frosted-glass card
           - Stacking queue (max 3 visible, FIFO drain)
           - 4-corner positioning, branding footer, progress bar
           - Dark mode via @media(prefers-color-scheme:dark) CSS only
           - Tag dedup (replaces existing notification with same tag)

  Type 1:  wrapper-v2.js → createWrapper()
           - Template compiled via: new Function(`return ${template.compiled_text}`)()
           - Wrapped in a dismissible container with auto-close

  Type -1: Web Push API → Service Worker
           - Handled by sw.js, uses push-notification.js for subscription

  Type "0"   →  Glassmorphism HTML (the ONLY in-app renderer)
  Type "1"   →  Custom Template (user JS inside glass container)
  Type "2"   →  Backward compat — maps to Type 0 (same glassmorphism)

  ⚠ Old toast/variant system REMOVED in v0.0.3.
    All in-app notifications use showHtmlNotification() exclusively.


────────────────────────────────────────────────────────────────────────────────
  RENDERING ENGINE (unified)
────────────────────────────────────────────────────────────────────────────────

  All types:  html-notification.js → showHtmlNotification()
              - Glassmorphism frosted-glass card for everything
              - Stacking queue (max N visible, FIFO drain)
              - 4-corner positioning, branding footer, progress bar
              - Explicit dark mode via `theme` param (.pn-dark class)
              - Tag dedup with smooth crossfade animation
              - Icon prefetching via <link rel="prefetch">
              - Hover-pause timer per card
              - Max 3 buttons with event dispatch support

  Type 0/2:  Full SDK-built content (title, body, icon, media, buttons)
  Type 1:    customContent option — user-provided HTMLElement inside
             glass container. SDK provides container chrome:
             header, close button, branding, progress bar, stacking,
             auto-close, dismiss tracking.
  Type -1:   Service Worker (unchanged from v0.0.2)
