SDK v0.0.3

Pinglet Notification Demo

All 4 notification types in action. Click any card to fire a live notification via the backend API.
Types 0, 1, 2 arrive via SSE → rendered in-page. Type -1 triggers a browser push via Service Worker.

TYPE 0 In-App Toast Notification

Classic slide/fade/zoom toast rendered via createVariant() + renderToast(). Supports title, description, buttons, icon, logo, media, dark mode.

0-A

Minimal — Title Only

Simplest toast with just a title.

0-B

Full — Title + Desc + Icon

Title, description, and a small icon.

0-C

Image Media

Toast with a full-width image banner.

0-D

Video Media

Toast with an embedded muted video.

0-E

With Buttons

Redirect + reload + event action buttons.

0-F

Logo + Click URL

Logo image, body click opens a link.

0-G

Stacking — Fire 5 at once

Fires 5 toasts rapidly to test stack behavior.

TYPE 2 Glassmorphism HTML Notification

Glassmorphism card rendered via showHtmlNotification(). Stacking queue, branding footer, progress bar, 4-corner positioning, dark mode, media, max 3 buttons.

2-A

Title Only

Minimal glassmorphism card, just a title.

2-B

Full — Icon + Body + Badge

Icon, body text, badge, and relative time.

2-C

Image Banner

Full-width image below the content.

2-D

Video Media

Embedded video player in the card.

2-E

3 Buttons — Redirect / Event / Close

Max 3 buttons with different actions.

2-F

Custom Event Button

Button dispatches a CustomEvent on window.

2-G

All 4 Corners

Shows one notification in each corner.

2-H

No Auto-Close (requireInteraction)

Stays until user dismisses. No progress bar.

2-I

Stacking — Fire 6 at once

Max 3 visible, rest queued and dequeued on dismiss.

2-J

Force Dark Theme

Explicitly dark mode regardless of system.

2-K

Custom Branding

Custom branding HTML at the bottom.

2-L

Tag Dedup — Replace

Same tag replaces existing notification.

TYPE -1 Browser Push Notification

Sent via Web Push API → Service Worker. Works when tab is backgrounded or closed. Requires push permission. Note: these go through BullMQ queue on backend.

-1-A

Basic Push

Title + body, appears in OS notification tray.

-1-B

Push with Icon + Badge

Custom icon and badge overlay.

-1-C

Push with Large Image

Big image shown in the push notification.

-1-D

Push with Actions

2 action buttons in native push (max 2).

-1-E

Silent Push

No sound or vibration, stealth notification.

TYPE 1 Custom Template Notification

Uses a pre-built server-stored template referenced by template_id. The template's compiled JS is executed with the custom_template data passed in. Rendered via createWrapper().

1-A

Template with Data

Sends custom_template data to a stored template.
